The motivation of proposing this benchmark:

Previous SoC test scheduling datasets, such as the ITC’02 benchmark,  only  focus  on  scan  tests  and  do  not  provide 
data  for test items  other than  scan tests. In  addition,  since the ITC’02 benchmark does not offer  specific circuits, it is 
unscalable. This creates the problem of a lack of standard test datasets for researchers aiming to study the joint scheduling 
of multiple  test  items, prompting us  to  construct  a  custom test dataset. 

MIST refers to Multi-Item SoC Test.
